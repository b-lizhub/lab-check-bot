🤖 Lab Check Bot

This Lab Check Bot verifies that you've completed the lab steps for a technical lab. All you need is the link to the lab's GitHub repo, and then follow the steps.

How to use this tool
  1. Choose a lab or paste a GitHub lab URL. You can pick a lab from the solution-area menu, or paste the published GitHub repo page for the lab.
  2. Load the lab and review the detected requirements. After the lab loads, the bot updates the sign-in area to show which extra platform inputs are Required and which are Not applicable for that specific lab.
  3. Sign in with a Global Administrator account in the tenant you want to check. All checks run in your browser — no data leaves your machine.
  4. First time in a tenant? Expand "First time in this tenant? Grant admin consent" and click 🔓 Grant admin consent for this tenant. A GA accepts the read-only Graph permissions once per tenant; after that, every user in that tenant just signs in.
  5. Add only the platform tokens the loaded lab requires. If the lab needs GitHub or Azure DevOps, the PAT panel will mark that input as Required. If the lab does not use that platform, it will show Not applicable. For Azure resources (App Service, Storage, Key Vault, OpenAI, etc.), the bot uses your signed-in Microsoft identity — you'll see a one-time 🔓 Grant Azure access button in the results if consent is needed.
  6. Click "▶ Run checks." The bot reads the lab markdown, detects the portals and named resources it needs to verify, then queries each reachable service to see what's already in place.
  7. Review results. The bot acts like a security reviewer: every row shows PASS / FAIL / EXPECTED MISSING / WARN / NOT VALIDATED / SKIP plus what was checked, the evidence found, why it reached that conclusion, what to do next, and the lab step it maps to. NOT VALIDATED means the outcome lives somewhere the browser can't reach (on-prem AD, local audit policy, the Defender portal) — follow the manual verification steps shown.

Copilot Studio agent verification and Azure resource verification each require one extra one-time admin consent in the tenant. GitHub and Azure DevOps checks use Personal Access Tokens that stay in your browser's localStorage, and the bot now marks them per lab as required or not applicable. The lab text never leaves your browser; the bot only calls api.github.com, raw.githubusercontent.com, graph.microsoft.com, management.azure.com, dev.azure.com, and (optionally) Dataverse for your tenant.
