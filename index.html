🤖 Lab Check Bot

This Lab Check Bot verifies that you've completed the lab steps for a technical lab. All you need is the link to the lab's GitHub repo, and then follow the steps.

How to use this tool
  1. Paste a GitHub lab URL. Anything works: a repo root, a /tree/main/docs/... folder, a single .md file, or a published GitHub Pages site (*.github.io/...).
  2. Sign in with a Global Administrator account in the tenant you want to check. All checks run in your browser — no data leaves your machine.
  3. First time in a tenant? Expand "First time in this tenant? Grant admin consent" and click 🔓 Grant admin consent for this tenant. A GA accepts the read-only Graph permissions once per tenant; after that, every user in that tenant just signs in.
  4. (Optional) Add platform tokens. If the lab uses GitHub or Azure DevOps, expand "Optional: GitHub & Azure DevOps access tokens" and paste a PAT. For Azure resources (App Service, Storage, Key Vault, OpenAI, etc.), the bot uses your signed-in Microsoft identity — you'll see a one-time 🔓 Grant Azure access button in the results if consent is needed.
  5. Click "▶ Run checks." The bot reads the lab markdown, detects the portals it touches (Copilot Studio, Intune, Entra, Azure, GitHub, Azure DevOps) and any named items (agents, pools, resources, repos), then queries each service to see what's already in place.
  6. Review results. Each row is PASS / WARN / FAIL / SKIP. For any FAIL or WARN, expand 📖 What the lab says to do to see the exact lab step(s) you were supposed to complete.

Copilot Studio agent verification and Azure resource verification each require one extra one-time admin consent in the tenant. GitHub and Azure DevOps checks use Personal Access Tokens that stay in your browser's localStorage. The lab text never leaves your browser; the bot only calls api.github.com, raw.githubusercontent.com, graph.microsoft.com, management.azure.com, dev.azure.com, and (optionally) Dataverse for your tenant.
